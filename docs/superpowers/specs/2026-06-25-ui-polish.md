# UI Polish — Design Spec

## Changes

### 1. TimePreferenceToggle — visual pill redesign
Replace the collapsed "+" link with three always-visible pills: **Now · Arrive by · Depart at**. "Now" selected by default (no time preference stored). Tapping "Arrive by" or "Depart at" reveals a time input inline below the pills. Active pill: dark filled (zinc-900 bg, white text). Inactive: outlined (zinc-200 border, zinc-700 text). Saves/clears via existing `POST /api/session/[id]/time` — same API, new UI.

### 2. StationCard — journey time emphasis + prominent average + rose "Show map"
- "X mins total" per person: `text-base font-bold text-zinc-900` (was small/medium)
- Average time: pulled out of the grey footer row into a standalone rose-tinted stat badge above the footer, clearly labelled "Avg. X mins"
- "Show map" button: small rose pill (`bg-rose-600 text-white rounded-full px-3 py-1 text-xs font-semibold`) replacing the underline link

### 3. LocationsForm — "+ Add another" pill bubble
Replace underline link with a rounded-full pill: light rose background (`bg-rose-50`), rose border (`border-rose-300`), rose text (`text-rose-600`), padding `px-4 py-2`.

### 4. Home page — mini example section
New section between "How it works" and FAQ: **"Here's what it looks like"** heading, hardcoded fake result card showing:
- 3 people: Tess from Brixton (22 mins), Sam from Shoreditch (24 mins), Jo from Ealing (26 mins)
- Result: Oxford Circus — "Longest journey: 26 mins"
- Styled like the real results (rose accent, journey time rows) but static, no API calls
- Small "Example only" label so it's clearly not a real search

## Constraints
- Mobile-first, 375px+
- No new dependencies
- No automated tests — verified manually in browser
- Commit messages without Co-Authored-By trailer
